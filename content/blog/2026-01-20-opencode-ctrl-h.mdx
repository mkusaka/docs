## まとめ

- Terminal.app等はCtrl+Hを`^H(0x08)`として送るが、OpenCodeのデフォルトkeybinds（`backspace,shift+backspace`）に`ctrl+h`が含まれないため文字削除できない
- `~/.config/opencode/opencode.json`の`input_backspace`に`ctrl+h`を足せば解決
- iTerm2/Ghostty等はkitty keyboard protocol等の拡張で修飾情報付きで送信できるため、端末によって挙動が異なる

## 解決方法

`~/.config/opencode/opencode.json`：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "keybinds": {
    "input_backspace": "backspace,shift+backspace,ctrl+h"
  }
}
```
